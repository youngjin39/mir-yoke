# ADR-09 — execution_backend frontmatter (sanitized summary)

Status: accepted (upstream)

Sanitized summary of the upstream ADR for fork users. The full decision
record lives in the upstream private harness; this stub captures the
contract that public-template fork users need to satisfy.

## Decision

`execution_backend` is the single declarative surface that names which
runtime executes an agent's body. Every agent frontmatter under
`.claude/agents/<slug>.md` must declare it:

```yaml
execution_backend: claude   # body executed by the Claude session
execution_backend: codex    # body executed by Codex CLI subprocess
```

No other surface (TOML, JSON, hook, body header prose) is authoritative.
`tools.agent_loader --mode=strict` validates the declaration.

## Codex agents

Three agents in this template declare `execution_backend: codex`:
- `codex-final-reviewer` — read-only review.
- `executor-agent` — code/TDD/review execution.
- `pipeline-validator` — read-only data-pipeline schema audit.

The Codex CLI subprocess invocation pattern is documented in
`.claude/agents/executor-agent.md` (the `perl -e 'alarm <N>; exec
@ARGV' codex exec ...` shape).

## Mirror artifacts

`.codex/agents/<slug>.toml` files mirror agent definitions for Codex
session consumption. They are regenerated by
`scripts/generate_codex_derivatives.sh` from the `.md` source. The
TOML files do NOT carry `execution_backend` — the `.md` frontmatter is
the only source of truth.
