# Mir Yoke — Harness Template Instructions

## Outcome and completion

- Maintain a public, repository-agnostic Claude/Codex harness template that adopters can fit to
  their own product without inheriting private fleet policy or unnecessary ceremony.
- Finish when the requested template outcome works for a clean adopter, the smallest relevant
  evidence passes, and changed scope, residual issues, and risks are explicit.
- Stop for user direction only when no safe in-scope path remains, authority is missing, or current
  sources contain a material conflict.

## Bootstrap and sources

- After cloning, replace placeholders in `.mir/repo-profile.toml`, run `./setup.sh`, initialize
  memory with `uv run mir migrate up`, and verify paths with
  `uv run python scripts/verify_context_paths.py`.
- `.mir/repo-profile.toml` is canonical for repository identity, role policy, protected paths, and
  execution boundaries. SessionStart supplies its compact baseline.
- For substantial repository-dependent work, make one task-scoped
  `uv run mir context pull "<query>" [--path <target>] [--risk low|normal|high]`; expand only for
  missing, stale, or conflicting evidence. Do not preload plans, projections, or history.
- `.mir/memory.db` is canonical memory. `docs/memory-map.md` and `tasks/lessons.md` are generated
  projections; do not hand-edit generated regions.

## Authority and safety

- Read, review, and status requests are non-mutating. Change, fix, or build requests authorize
  in-scope repository edits and relevant verification.
- Get explicit direction before destructive actions, credential or secret access, external writes
  or messages, protected-scope mutation, or material scope expansion.
- Keep the public surface generic and sanitized. Repository-specific workflow, hooks, agents, and
  optional capabilities remain adopter-owned unless a shared correctness or safety fix requires a
  template change.
- Edit canonical sources first: `CLAUDE.md`, `.claude/agents/`, and `.claude/skills/`. Regenerate
  `AGENTS.md`, `.codex/`, and `.agents/`; do not hand-edit them.

## Execution and evidence

- Use the least process that preserves the outcome and boundaries. Direct bounded work is valid;
  delegate only when isolation, parallelism, independent review, or context economy materially
  helps. Raw `codex exec` is not an allowed delegation path.
- Run the smallest check that can fail for changed behavior. Use a persisted plan, TDD ledger,
  worktree, or broader suite only when risk or restartability warrants it.
- Detailed development and closeout procedures are on demand in `.ai-harness/` and
  `docs/harness-engineering/`.
- On an explicit closeout request, refresh `tasks/handoffs/session-handoff-LATEST.md` through
  `.claude/hooks/session-end.sh`; do not create a competing summary.

Commands: `uv run pytest`, `uv run ruff check`, `uv run mir --help`.
User-facing language follows the adopter's convention. Internal docs, code, commits, and handoffs
are English.
